# 股票技术指标交互分析工具 · 产品设计文档

> **版本**：v1.0 PRD
> **日期**：2026-07-04
> **定位**：一个纯前端（HTML + JS）的技术指标交互分析工具，支持多股票切换、四指标参数可调、实时重算重绘。

---

## 一、产品概述

### 1.1 目标用户

| 用户类型 | 需求 |
|:---|:---|
| 量化初学者 | 直观理解 RSI/MACD/布林带/KDJ 的参数变化对指标形态的影响 |
| BA 工作坊学员 | 在课堂作业中快速对比不同股票的技术指标信号 |
| 个人投资者 | 对锂业三股（天齐/赣锋/盐湖）进行技术面快速研判 |

### 1.2 核心价值

> **"调参即见果"** — 用户拖动滑块修改参数，图表实时重算重绘，无需写代码即可探索不同参数组合下指标的表现差异。

### 1.3 与现有产出的关系

| 已有产出 | 本工具的角色 |
|:---|:---|
| `TASK02/tianqi_indicators.html`（静态面板） | 固定参数、固定股票、静态 PNG |
| `TASK02/scripts/calc_indicators.py`（Python 脚本） | 需命令行运行，改参数需改代码 |
| **本工具** | **浏览器中交互操作，选股票 + 调参数 + 实时重绘** |

```
打开页面
  → 左侧选择股票（天齐/赣锋/盐湖）
```

## 三、技术架构

### 3.1 整体架构（纯前端）

```
┌─────────────────────────────────────────────┐
│              浏览器（单个 HTML 文件）           │
│                                             │
│  ┌─────────┐   ┌──────────┐   ┌─────────┐  │
│  │ 数据层   │   │ 计算层    │   │ 渲染层   │  │
│  │ JSON嵌入 │──→│ JS指标引擎│──→│ ECharts │  │
│  │ 3只股票  │   │ RSI/MACD │   │ 4张图表  │  │
│  │          │   │ BOLL/KDJ │   │         │  │
│  └─────────┘   └──────────┘   └─────────┘  │
│        ↑              ↑            ↑        │
│        └────── 事件层（滑块/下拉）──┘        │
│                                             │
│  数据流：参数变化 → 重算 → 重绘（无网络请求） │
└─────────────────────────────────────────────┘
```

**关键决策：数据预嵌入，无后端**

| 方案 | 选择 | 理由 |
|:---|:---|:---|
| 数据获取 | **预嵌入 JSON** | 三股数据量小（各~62KB），直接嵌入 HTML，避免 CORS 和 API 调用 |
| 指标计算 | **纯 JS 前端计算** | 移植 Python 逻辑到 JS，无需后端 |
| 图表渲染 | **ECharts（CDN）** | 成熟、交互丰富、中国股市配色方便 |
| 部署 | **GitHub Pages** | 与现有项目一致，静态托管 |

### 3.2 文件结构

```
TASK02/
├── indicator_tool/
│   ├── index.html              ← 主页面（入口）
│   ├── css/
│   │   └── style.css           ← 样式
│   ├── js/
│   │   ├── data.js             ← 三股 JSON 数据（嵌入）
│   │   ├── indicators.js       ← 四指标计算引擎（RSI/MACD/BOLL/KDJ）
│   │   ├── charts.js           ← ECharts 图表渲染
│   │   ├── signals.js          ← 信号判定 + 共振逻辑
│   │   └── app.js              ← 主控制器（事件绑定 + 数据流）
│   └── README.md               ← 工具说明
```

### 3.3 指标计算引擎（JS 移植说明）

将 `calc_indicators.py` 中的四个函数移植为 JS：

| 指标 | Python 函数 | JS 函数 | 参数 |
|:---|:---|:---|:---|
| RSI | `calc_rsi(prices, 14)` | `calcRSI(prices, period)` | period |
| MACD | `calc_macd(prices, 12, 26, 9)` | `calcMACD(prices, fast, slow, signal)` | fast, slow, signal |
| 布林带 | `calc_boll(prices, 20, 2)` | `calcBOLL(prices, period, nbdev)` | period, nbdev |
| KDJ | `calc_kdj(highs, lows, closes, 9, 3, 3)` | `calcKDJ(highs, lows, closes, n, m1, m2)` | n, m1(m2) |

> 移植要点：Python 的 `None` → JS 的 `null`；Python 的 `math.isnan` → JS 的 `isNaN`；其余逻辑完全一致。

## 五、信号判定逻辑

### 5.1 单指标信号

| 指标 | 买入信号 | 卖出信号 | 中性 |
|:---|:---|:---|:---|
| RSI | RSI < 30（超卖回升） | RSI > 70（超买回落） | 30 ≤ RSI ≤ 70 |
| MACD | 金叉（DIF上穿DEA） | 死叉（DIF下穿DEA） | 无交叉 |
| 布林带 | 触及下轨（<-2σ） | 触及上轨（>+2σ） | 中轨附近 |
| KDJ | 金叉 + K<20 超卖 | 死叉 + K>80 超买 | 20≤K≤80 |

### 5.2 多指标共振评分

```
买入得分 = count(四指标中买入信号数)    // 0~4
卖出得分 = count(四指标中卖出信号数)    // 0~4
中性得分 = 4 - 买入 - 卖出

综合研判:
  买入 ≥ 3  →  强买入
  买入 = 2  →  偏多
  卖出 ≥ 3  →  强卖出
  卖出 = 2  →  偏空
  否则      →  中性震荡
```

## 九、风险与对策

| 风险 | 影响 | 对策 |
|:---|:---|:---|
| ECharts CDN 加载慢 | 页面白屏 | 提供本地 echarts.min.js fallback |
| 滑块频繁触发导致卡顿 | 体验差 | 加 debounce（200ms 防抖） |
| 参数极端值计算异常 | NaN/Infinity | 输入校验 + 最小值限制 |
| 数据量增大（多股/多年） | 加载慢 | 数据分片加载或改用后端 API |
