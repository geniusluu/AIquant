# TASK02 · 锂业三股规范化取数任务规划

> **目标**：为天齐锂业、赣锋锂业、盐湖股份三只典型锂业股票建立一套**可复用、可审计、规范化**的取数工作流。
> 本文件是任务执行的总纲，具体机器可读参数规范见同目录 `stock_data_spec.json`，人类可读说明见 `stock_data_spec.md`。

---

## 〇、与 TASK1-2 的关系

| 维度 | TASK1-2（v1.0） | TASK02（v2.0） |
|:---|:---|:---|
| 定位 | 首次取数 + 规范初稿 | 规范化取数体系定型 |
| 复权处理 | 未明确，仅存 raw | 明确 raw/qfq/hfq 策略，默认前复权 |
| 除权除息 | 事后补查 | 纳入正式存储 + 事件日志 |
| 交易日历 | 文档提及未执行 | 自动化校验断档 |
| Token 加载 | 环境变量 | MCP 配置优先 + 环境变量 fallback |
| 频率限制 | 粗粒度 | 按接口细化（A股复权因子 65s 间隔） |
| 增量更新 | 无 | 定义全量/增量两种模式 |
| 数据血缘 | 无 | 每批数据附带元数据（来源/时间戳/行数） |

TASK02 复用 TASK1-2 已落地的原始数据（`TASK1-2/raw_data/`），在其基础上补全复权价、事件日志，并定型规范。

## 二、工作步骤总览

```
Step 1  确认标的元数据（代码/交易所/币种/市场）
Step 2  定义字段集、时间范围与复权策略
Step 3  配置数据源与认证（Tushare token / 东方财富备选）
Step 4  编写通用取数脚本（读 spec → 批量取数 → 落地）
```

## 三、详细工作步骤

### Step 1 · 确认标的元数据

| 项目 | 说明 |
|:---|:---|
| 目的 | 锁定每只股票的唯一标识 `ts_code`，区分 A股/港股 |
| 产出 | `stock_data_spec.json` → `instruments` 段 |
| 数据来源 | Tushare `stock_basic` / `hk_basic` 接口核对 |
| ⚠️ 注意 | 港股代码 `XXXXX.HK`（5位）；A股 `XXXXXX.SZ/SH`（6位）；盐湖股份无港股 |

### Step 2 · 定义字段集与复权策略

| 项目 | 说明 |
|:---|:---|
| 目的 | 统一 schema，明确复权处理方式 |
| 产出 | `stock_data_spec.json` → `fields` / `adjustment` 段 |
| 核心字段 | `trade_date, open, high, low, close, vol, amount, pct_chg` |
| 复权策略 | 默认同时存储 **raw（不复权）** + **qfq（前复权）**；横向比较用 qfq |
| 时间范围 | 近 365 天滚动窗口（可切固定模式） |
| 频率 | 日线（daily），可扩展 weekly/monthly |

### Step 3 · 配置数据源与认证

| 项目 | 说明 |
|:---|:---|
| 主数据源 | Tushare Pro（`daily` / `hk_daily` / `adj_factor` / `hk_adjfactor` / `dividend`） |
| 备选源 | 东方财富（免登录，字段需映射） |
| Token 加载 | 优先读 `~/.workbuddy/mcp.json`，fallback 环境变量 `TUSHARE_TOKEN` |
| 切换策略 | Tushare 连续失败 3 次自动切备选源 |

### Step 4 · 编写通用取数脚本

| 项目 | 说明 |
|:---|:---|
| 设计要点 | ① 读取 spec.json 的 instruments 循环取数；② 按后缀自动选接口；③ 频率限制自动间隔 |
| 接口映射 | `.SZ/.SH → daily`；`.HK → hk_daily` |
| 依赖 | 仅 Python 标准库（urllib/json/csv），与项目保持一致 |
| 产出 | `scripts/fetch_all.py` |

### Step 5 · 获取复权因子与除权除息事件

| 项目 | 说明 |
|:---|:---|
| 目的 | 构建前复权价 + 记录公司行动事件 |
| A股接口 | `adj_factor`（⚠️ 频率限制：1次/分钟，间隔需 ≥65s） |
| 港股接口 | `hk_adjfactor`（限制较宽，间隔 1s） |
| 除权除息 | `dividend` 接口，记录分红/送股/转增 |
| 产出 | `raw_data/{id}_adj.json` + `除权除息事件日志.json` |

### Step 6 · 执行取数并落地

| 项目 | 说明 |
|:---|:---|
| 存储格式 | JSON（结构化）+ CSV（UTF-8 BOM，Excel 直开） |
| 命名规则 | `{instrument_id}_data_{adjust}.{ext}`，如 `ganfeng_a_data_qfq.csv` |
| 排序 | `trade_date` 升序 |
| NaN 处理 | JSON 中 `NaN → null` |
| 位置 | `TASK02/raw_data/` |

### Step 7 · 数据质量校验

| 校验项 | 规则 | 处置 |
|:---|:---|:---|
| 行数 | A股 230~252；港股 235~252 | 超范围告警 |
| 缺失值 | OHLCV 不允许空 | 告警 + 人工复核 |
| 涨跌异常 | `pct_chg` 超 ±20% | 标记复核（排除除权后） |
| 日期连续性 | 对 `trade_cal` 检查断档 | 停牌日期记录 |

### Step 8 · 三股横向对齐

| 项目 | 说明 |
|:---|:---|
| 基准 | 三只 A股交易日交集 |
| 产出 | `三股对齐数据.csv`（close + pct_chg 按列拼接） |
| 用途 | 收益率对比、相关性矩阵、标准化走势叠图 |

### Step 9 · 数据血缘报告

每批取数完成后生成 `数据血缘报告.json`，记录：
- 取数时间戳（UTC+8）
- 数据源版本（Tushare 接口名）
- 每只标的的行数、日期范围、校验结果
- 复权因子跳变次数、除权除息事件数

## 五、风险与注意事项

1. **Tushare 频率限制**：
   - `adj_factor`（A股复权因子）：**1次/分钟**，三股需间隔 65s，全程约 4 分钟。
   - `hk_daily` / `hk_adjfactor`：间隔 ≥1s 即可。
   - `daily`（A股行情）：间隔 ≥200ms。
2. **复权一致性**：横向比较三股走势时**必须统一用前复权（qfq）**，否则除权日会产生虚假跳变。
3. **币种**：A股 CNY，港股 HKD，横向比较需标注或做汇率换算。
4. **盐湖股份特殊性**：2024-2025 年间曾有重大事项停牌，取数时需确认连续性；ST 摘帽历史需关注。
5. **增量更新**：后续更新数据时，优先用增量模式（仅取上次截止日之后的新数据），减少 API 调用。

```
第一轮（约 5 分钟）：
  Step 1-3 → 确认 spec，配置 token
  Step 4   → 取 A股 + 港股 raw 行情（快，无严格限制）

  Step 5   → 取港股复权因子（快）+ A股复权因子（慢，65s间隔）

  Step 6-9 → 生成 qfq、校验、对齐、血缘报告
```
